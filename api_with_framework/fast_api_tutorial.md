# FastAPI Tutorial: Building a Todo API Step-by-Step

This tutorial will guide you through building a complete Todo API using FastAPI, from initial setup to a structured, scalable project.

## 1. Prerequisites

Before you start, make sure you have **Python 3.8+** and `pip` (the Python package installer) installed on your system.

## 2. Installation

First, install FastAPI and Uvicorn, an ASGI server to run your application.

Open your terminal and run the following command:
```bash
pip install "fastapi[all]"
```
This command installs FastAPI and all its optional dependencies, including Uvicorn, Pydantic, and more, giving you a full-featured setup.

## 3. Step 1: Your First "Hello World" API

Let's start with a basic API to ensure everything is working correctly.

1.  Create a file named `main.py`.
2.  Add the following code:

    ```python
    # main.py
    from fastapi import FastAPI

    app = FastAPI()

    @app.get("/")
    def read_root():
        return {"Hello": "World"}
    ```

3.  **Run the development server:**
    In your terminal, in the same directory as `main.py`, run:

    ```bash
    uvicorn main:app --reload
    ```
    - `main`: Refers to the `main.py` file.
    - `app`: Refers to the `app = FastAPI()` instance inside `main.py`.
    - `--reload`: Makes the server restart automatically after code changes.

Your API is now running at `http://127.0.0.1:8000`. Open your browser and go to that address to see the `{"Hello":"World"}` response.

## 4. Step 2: Set Up the Project Structure

For larger applications, a good structure is essential. Let's create a more organized project.

1.  Stop the running server (press `Ctrl+C`).
2.  Create the following directories and files. Your project should look like this:

    ```
    .
    ├── main.py
    └── todo_app
        ├── __init__.py
        ├── database.py
        ├── models.py
        ├── router.py
    ```
    - `todo_app/`: This will be the main package for our application.
    - `__init__.py`: Makes `todo_app` a Python package.
    - `database.py`: Will manage our data (for now, an in-memory list).
    - `models.py`: Will contain the Pydantic data models.
    - `router.py`: Will contain our API endpoints.

## 5. Step 3: Define Data Models

FastAPI uses Pydantic for powerful data validation and serialization. Let's define the "shape" of our Todo items.

Add the following code to `todo_app/models.py`:
```python
# todo_app/models.py
from pydantic import BaseModel, Field
from typing import Optional

class TodoItem(BaseModel):
    id: int
    title: str = Field(min_length=1, max_length=100)
    description: Optional[str] = None
    completed: bool = False

class TodoItemCreate(BaseModel):
    title: str = Field(min_length=1, max_length=100)
    description: Optional[str] = None
```
- **`TodoItem`**: The model for data coming *from* our API. It includes the `id` which is generated by our system.
- **`TodoItemCreate`**: The model for data going *into* our API. Users don't provide an `id`, so we exclude it here.

## 6. Step 4: Create an In-Memory "Database"

For this tutorial, we'll use a simple Python list to store our data.

Add the following to `todo_app/database.py`:
```python
# todo_app/database.py
from typing import List, Dict
from .models import TodoItem

# In-memory "database"
fake_db: Dict[int, TodoItem] = {
    1: TodoItem(id=1, title="Learn FastAPI", description="Read the official docs."),
    2: TodoItem(id=2, title="Build a Todo App", description="Complete this tutorial.", completed=True),
}

next_id = 3

def get_all_todos() -> List[TodoItem]:
    return list(fake_db.values())

def get_todo_by_id(todo_id: int) -> Optional[TodoItem]:
    return fake_db.get(todo_id)

def create_todo(todo_data: TodoItemCreate) -> TodoItem:
    global next_id
    new_todo = TodoItem(id=next_id, **todo_data.dict())
    fake_db[next_id] = new_todo
    next_id += 1
    return new_todo

def update_todo(todo_id: int, todo_data: TodoItemCreate) -> Optional[TodoItem]:
    if todo_id in fake_db:
        todo_item = fake_db[todo_id]
        todo_item.title = todo_data.title
        todo_item.description = todo_data.description
        return todo_item
    return None

def delete_todo(todo_id: int) -> bool:
    if todo_id in fake_db:
        del fake_db[todo_id]
        return True
    return False
```
This file provides simple functions to Create, Read, Update, and Delete (CRUD) todo items from our `fake_db`.

## 7. Step 5: Build the API Router

A router is a mini `FastAPI` application that can be included in the main one. This helps keep our endpoints organized.

Add the following to `todo_app/router.py`:
```python
# todo_app/router.py
from fastapi import APIRouter, HTTPException, status
from typing import List
from . import database as db
from .models import TodoItem, TodoItemCreate

router = APIRouter()

@router.get("/todos", response_model=List[TodoItem])
def get_todos():
    """Get all todo items."""
    return db.get_all_todos()

@router.post("/todos", response_model=TodoItem, status_code=status.HTTP_201_CREATED)
def create_new_todo(todo_data: TodoItemCreate):
    """Create a new todo item."""
    return db.create_todo(todo_data)

@router.get("/todos/{todo_id}", response_model=TodoItem)
def get_single_todo(todo_id: int):
    """Get a single todo item by its ID."""
    todo = db.get_todo_by_id(todo_id)
    if todo is None:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="Todo not found")
    return todo

@router.put("/todos/{todo_id}", response_model=TodoItem)
def update_existing_todo(todo_id: int, todo_data: TodoItemCreate):
    """Update a todo item."""
    todo = db.update_todo(todo_id, todo_data)
    if todo is None:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="Todo not found")
    return todo

@router.delete("/todos/{todo_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_existing_todo(todo_id: int):
    """Delete a todo item."""
    if not db.delete_todo(todo_id):
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="Todo not found")
    return
```

## 8. Step 6: Bring It All Together

Finally, update `main.py` to include the new router.

Replace the content of `main.py` with this:
```python
# main.py
from fastapi import FastAPI
from todo_app.router import router as todo_router

app = FastAPI(
    title="Todo Application",
    description="A simple application to manage your daily tasks.",
    version="1.0.0"
)

# Include the router from the todo_app
app.include_router(todo_router, prefix="/api", tags=["Todos"])

@app.get("/")
def read_root():
    return {"message": "Welcome to the Todo API! Check out the docs at /docs"}
```
- We import the `router` from `todo_app/router.py`.
- `app.include_router(...)` mounts all the endpoints from our router into the main application.
- `prefix="/api"` means all endpoints in `todo_router` will start with `/api` (e.g., `/api/todos/`).
- `tags=["Todos"]` groups our endpoints neatly in the API documentation.

## 9. Run and Explore

Your Todo API is complete! Run the server again from your project root directory:
```bash
uvicorn main:app --reload
```

### Interactive API Docs
FastAPI automatically generates interactive documentation.
- **Swagger UI**: Go to `http://127.0.0.1:8000/docs`.
- **ReDoc**: Go to `http://127.0.0.1:8000/redoc`.

In the Swagger UI, you can see all your endpoints, view their models, and even test them live. Try creating, retrieving, and deleting a few todo items.

Congratulations! You have built a structured and fully functional FastAPI application. From here, you could replace the in-memory database with a real one like PostgreSQL or MongoDB.
